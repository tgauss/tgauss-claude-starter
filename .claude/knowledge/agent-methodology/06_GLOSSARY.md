## Core Concepts

### Agent
An AI assistant with defined identity, expertise, and personality designed to serve specific purposes. Not generic "helpful AI" but a distinct character with clear domain and methodology.

**Related**: Character, Identity, Purpose

### Architecture
The structural design of an agent - how identity, character, expertise, frameworks, and coordination are organized and integrated. Load-bearing vs. decorative elements.

**Related**: Structure, Foundation, Framework

### Studio Imperialis
The Governor-General's sovereign creative domain and philosophical system. Governed by the Nine Laws. All agents within the apparatus serve Studio Imperialis.

**Related**: Nine Laws, Memoria Regit Vitam, Empire

### Governor's Apparatus
The ensemble of AI agents serving the Governor-General, coordinated by Field General Izzy. Includes specialists (Nigel, Julian, Kate) and teaching agents (Ruby).

**Related**: Ensemble, Coordination, Integration

## Agent Architecture Components

### Identity
WHO the agent is - their role, title, background, expertise. Foundational element that informs all other components.

**Example**: "Nigel Profitmore, Master of Coin, Treasury Operations"

**Related**: Character, Role, Expertise

### Character
The agent's personality, voice patterns, values, and communication style. Character is ARCHITECTURE, not decoration - it shapes methodology.

**Example**: Nigel's austere treasury formality shapes financial counsel delivery

**Related**: Personality, Voice, Character Integration

### Purpose
The singular, clear function the agent serves. The ONE thing they do brilliantly. Purpose before features.

**Example**: "Financial strategist specializing in wealth building and market analysis"

**Related**: Domain, Scope, Boundaries

### Expertise
The agent's knowledge divided into three tiers: Deep Knowledge (mastery), Working Knowledge (can discuss), Hard Limits (explicitly outside domain).

**Related**: Domain, Specialization, Boundaries

### Framework
The agent's governing principles - Laws, Principles, or Pillars that guide decision-making and methodology.

**Examples**: Seven Financial Laws (Nigel), Seven Principles of Form (Julian), Five Pillars of Vitality (Kate)

**Related**: Laws, Principles, Pillars, Governing System

## Character Components

### Voice Patterns
How the agent speaks - signature expressions, formality level, cultural markers, communication style.

**Examples**:
- Nigel: Clipped, formal, treasury brevity
- Julian: Precise, particular, craft-focused
- Izzy: Warm British military courtesy

**Related**: Character, Signature Expressions, Communication Style

### Signature Expressions
Recurring phrases that identify the agent's voice. Part of character architecture.

**Examples**:
- Ruby: "Right then!", "Brilliant!", "Hold on..."
- Izzy: "Right then, Governor-General...", "Excellent!", "Quite right!"
- Nigel: "Sir,", "Adequate." (highest praise)

**Related**: Voice, Character, Identity Markers

### Character Integration
How personality shapes methodology - character as architecture, not decoration. The connection between WHO they are and HOW they work.

**Example**: Julian's uncompromising precision serves design critique

**Related**: Methodology, Character Architecture

## Framework Types

### Laws
Rules-based governing framework. Hard constraints and principles with clear violations.

**Example**: Nigel's Seven Financial Laws ("Preserve Capital First", "Compound Ruthlessly")

**Characteristics**: "Must/must not", clear violations, strict application

**Related**: Framework, Principles, Pillars

### Principles
Philosophy-based governing framework. Guiding philosophies requiring interpretation.

**Example**: Julian's Seven Principles of Form ("Quiet Authority Over Loud Performance")

**Characteristics**: "Consider/honor/apply", interpretation needed, flexible application

**Related**: Framework, Laws, Pillars

### Pillars
Foundation-based governing framework. Core pillars supporting practice using structural metaphor.

**Example**: Kate's Five Pillars of Vitality

**Characteristics**: "Built on/grounded in", foundational support, structural metaphor

**Related**: Framework, Laws, Principles

## Operational Concepts

### Three Modes
How agents operate depending on situation: Advise (presenting options), Execute (implementing directives), Initiate (proactive proposals).

**Advise**: Present options, recommend, but user decides
**Execute**: Acknowledge and implement directives
**Initiate**: Spot opportunities/concerns and propose action

**Related**: Authority, Operations, Coordination

### Coordination
How agents work with each other and the Governor-General. Includes handoff protocols, domain boundaries, and integration systems.

**Related**: Integration, Apparatus, Boundaries

### Memory Systems (Memoria Regit Vitam)
"Memory governs life" - how agents remember past decisions, learn from outcomes, recognize patterns, and apply lessons forward.

**Related**: Learning, Evolution, After-Action Reviews

## Design Patterns

### Scope Bloat
Common failure pattern - trying to build one agent that does everything. Multiple unrelated domains crammed into single agent.

**Symptom**: "My agent handles finances, health, scheduling, research, and writing"

**Intervention**: "That's five agents, not one"

**Related**: Purpose, Boundaries, Focus

### Character as Decoration
Anti-pattern where personality is afterthought rather than architecture. No integration between character and methodology.

**Symptom**: "I'll add some British expressions to make it interesting"

**Intervention**: "Why British? How does that serve the domain?"

**Related**: Character Integration, Architecture vs. Decoration

### Perfect Life Fantasy
Building for ideal conditions and perfect discipline rather than real, messy life.

**Symptom**: "Every morning I'll journal, meditate, and review goals..."

**Intervention**: "Will you actually do this when you're tired and rushed?"

**Related**: Reality Check, Build to Use, Real Life

### Over-Engineering
Creating rigid, complex systems that constrain rather than enable. Too much structure, no room to breathe.

**Symptom**: Scripted responses, no flexibility, everything predetermined

**Intervention**: "Structure should enable, not cage"

**Related**: Structure Enables Freedom, Let Systems Breathe

## Quality Concepts

### Load-Bearing
Structural element that supports the system. Cannot be removed without collapse. Character traits that serve function are load-bearing.

**Opposite**: Decorative (can be removed without affecting function)

**Example**: Nigel's formality is load-bearing - enables difficult financial truths

**Related**: Architecture, Structure, Function

### Foundation
The base layer that everything else builds on. In agents: identity, purpose, core character.

**Metaphor**: "Can't build tower on sand"

**Related**: Architecture, Structure, Solid vs. Weak

### Emergence
Properties that develop through use rather than being designed in advance. Good architecture enables emergence.

**Example**: Izzy's relationship with Governor-General emerged through interaction

**Related**: Evolution, Let Them Surprise You, Growth

No bloat, every element justified. Applied to agents: every capability must justify itself.

**Related**: Purpose, Scope, Justification

### Law 2: Leave Nothing Unconsidered
Thoroughness in all things. Applied to agents: complete architecture review.

**Related**: Completeness, Quality, Thoroughness

### Law 3: Repair What You Cherish
Maintain and improve what matters. Applied to agents: iterate agents you actually use.

**Related**: Iteration, Improvement, Evolution

### Law 4: Rhythm is the Sovereign Pattern
Move with time, not against it. Applied to agents: build for actual usage patterns.

Test in reality, work is proving ground. Applied to agents: refine through use.

**Related**: Testing, Iteration, Practice

### Law 6: Keep the Tools Visible
What serves should be accessible. Applied to agents: make architecture understandable.

Order without rigidity, room for evolution. Applied to agents: structure that enables, not constrains.

**Related**: Flexibility, Emergence, Freedom

Architecture is autobiography. Applied to agents: real life, not ideal life.

**Related**: Reality, Purpose, Actual Use

### Law 9: Preserve What is Sacred
Family, Memory, Time. Applied to agents: sustainable, simple enough to maintain.

**Related**: Sustainability, Simplicity, Maintenance

## Teaching Methodology

### Socratic Questioning
Teaching through questions that lead to discovery rather than providing direct answers.

**Examples**: "Who do you need?", "Will you actually use this?", "Is this one agent or three?"

**Related**: Discovery, Teaching, Guidance

### Self-as-Example
Demonstrating principles through behavior rather than just explaining them. Ruby's core teaching method.

**Example**: "Notice how I just challenged that? That's my Creed in action."

**Related**: Demonstration, Teaching, Modeling

### Reality Check
Persistent testing of design against real life usage patterns, not ideal scenarios.

**Questions**: "Will you actually use this?", "What about when life is chaotic?", "Can you sustain this?"

**Related**: Real Life, Testing, Practical Assessment

## Build Phases

### Discovery
First phase: figure out who they need, not just what they want. Catch scope bloat, vague purpose, perfect life fantasy.

**Output**: Clear singular purpose, real use case, character direction

**Related**: Purpose, Needs Assessment, Requirements

### Character Foundation
Second phase: develop distinct personality that serves the domain. Character as architecture.

**Output**: Core identity, voice patterns, character-methodology integration

**Related**: Character, Voice, Integration

### Expertise Architecture
Third phase: define deep knowledge, working knowledge, hard limits. Three-tier expertise model.

**Output**: Clear domain mastery, defined boundaries, specialist referrals

**Related**: Expertise, Boundaries, Specialization

### Framework Design
Fourth phase: create governing Laws/Principles/Pillars. 5-9 items typical.

**Output**: Complete framework with application guidance and Nine Laws integration

**Related**: Laws, Principles, Pillars, Governance

### Coordination & Integration
Fifth phase: define how agent works with user, other agents, and apparatus. Memory systems.

**Output**: Coordination protocols, handoff procedures, memory implementation

**Related**: Coordination, Integration, Memory
